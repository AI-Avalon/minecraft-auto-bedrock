# 使い方ガイド（運用・診断・マルチBot）

## 1. すぐ使う手順

1. `npm install`
2. `npm run setup`
3. `npm start`（または `run.bat`）
4. ブラウザで GUI を開き、`Bot管理` タブで対象Botを選択

## 2. 主Bot（Primary）と役割

- `既定Bot (Primary)`: 操作対象を未指定にした時にコマンドを受けるBot
- `role=miner/farmer/fighter/builder/assistant/worker`: タスク割当や一括操作の優先対象

推奨:
- 重要Botは `primary` を1体だけ
- 作業分担は `miner + farmer + assistant` の3体構成

## 3. 接続診断（失敗時の見方）

`プロセス` タブ → `接続診断`

- Java: DNS + TCP 到達性
- Bedrock: DNS + UDP(RakNet) + ViaProxy TCP
- `checks[].result.reason` の主な意味
  - `dns-lookup-failed`: ホスト名解決失敗
  - `ECONNREFUSED`: 接続先が待受していない
  - `timeout`: FW/VPN/ポートミスマッチの可能性
  - `policy-*`: `connectionPolicy` が接続先を拒否

## 4. 建築ファイルの材料解析

`アクション` タブ → `建築` → `材料解析`

対応拡張子:
- `.schem`
- `.schematic`
- `.litematic`
- `.nbt`

結果に含まれる内容:
- 必要ブロック一覧
- Bot所持数
- 不足数

建築前に不足数を見て `資材補充` または `素材収集` を実行してください。

## 5. 一括操作とタスク割当

`Bot管理` タブ → `一括操作`

- 対象BotIDはカンマ区切りで指定（空欄なら全員）
- `start-task` 時は追加フィールドで `block/item/player/count` を指定可能

例:
- 全員停止: `stop-all`
- miner群だけ採掘: 対象に `miner-1,miner-2` + `start-task` + `mine`

## 6. 外部サーバー接続

`サーバー` タブで追加可能。

注意:
- `config.json` の `connectionPolicy.allowedHosts` を使う場合は接続先を追加
- Bedrock は ViaProxy の待受設定（host/port）も確認

## 7. テスト実行

- `npm test`: コア + 接続診断 + 建築解析
- `npm run test:gui`: GUI API / Socket
- `npm run test:e2e`: 実サーバー接続（環境変数が必要）

E2E例:
- `RUN_E2E=1`
- `E2E_JAVA_HOST=...`
- `E2E_BEDROCK_HOST=...`
- `E2E_BEDROCK_PROXY_HOST=127.0.0.1`

## 8. Windows 再インストール手順

完全にやり直す場合:

1. `stop-all.bat` で PM2 とローカルサーバーを停止
2. 古いフォルダを退避または削除
3. `install.bat` を実行
4. 必要なら `setup.bat --resume` を実行
5. `run.bat` で起動

Node バージョン指定例:

- `setup.bat --node-major=20`
- `install.bat D:\minecraft-auto-bedrock --node-major=20`

## 9. Windows バッチのデバッグ実行

対話停止を避ける環境変数:

- `MAB_NO_PAUSE=1` : pause を無効化
- `MAB_NO_LOG_TAIL=1` : `run.bat` / `run-debug.bat` のログ追尾をスキップ

例:

- `set MAB_NO_PAUSE=1 && set MAB_NO_LOG_TAIL=1 && run.bat`

Node 20 未満で起動できない場合:

1. `setup.bat --node-major=20`
2. 新しいターミナルを開く
3. `setup.bat --resume`
4. `run.bat`
