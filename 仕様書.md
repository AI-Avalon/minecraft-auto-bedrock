ステップ1：ハイブリッド基盤と自動環境構築
このステップでは、Java/統合版の切り替えロジックと、必要なプロキシの自動取得を実装します。

指示 1: ハイブリッド基盤と自動セットアップの実装
マインクラフトのJava版と統合版（Geyser経由）の両方に対応し、設定一つで切り替え可能なMineflayerボットを作成します。
以下の機能を持つ package.json、config.json、setup.js を生成してください。

デュアルモード切り替え:

config.json の edition 設定（"java" または "bedrock"）に応じて接続方式を自動変更するロジック。

ViaProxy/Geyserの自動管理:

統合版モードの場合、Javaプロトコルを翻訳する最新の ViaProxy をGitHubから自動ダウンロードし 、ボット起動前にバックグラウンドで実行する機能。

OS判定: MacとWindowsのパス区切りや実行コマンドの差異を自動吸収する設計。

依存関係の網羅:

mineflayer, mineflayer-pathfinder, mineflayer-collectblock, mineflayer-tool, mineflayer-auto-eat, mineflayer-schem, prismarine-viewer, express, socket.io, pm2, simple-git, lowdb (記憶用)。

日本語注釈: すべてのコードに詳細な日本語コメントを付与してください。

ステップ2：記憶機能（空間メモリ）を備えた自律AI
このステップでは、ボットに「ものの場所を覚え、知能的に行動する」脳を与えます。

指示 2: 記憶機能（空間メモリ）を持つ自律AIの実装
src/bot.js を作成し、Mineflayerを使用して以下の「人間以上に賢い」機能を実装してください。

空間メモリ (Persistent Memory):

lowdb 等を使用し、発見したチェストの座標、中身の履歴、拠点の位置、死亡地点を memory.json に永続保存する 。

一度開いたチェストは「何が入っていたか」を記憶し、必要な資材を出す際に最適なチェストへ直行する。

高度な生存維持:

mineflayer-auto-eat での自動食事 。Java版ではオフハンド食いも考慮。

体力が危険域（10以下）になったら、記憶している「安全な拠点」へ即座に退避する。

採掘・建築・ツール管理:

mineflayer-collectblock での自動採取 。

mineflayer-tool で最強のエンチャントツールを自動選択 。

mineflayer-schem による .schem ファイルからの自動建築。資材が足りなければ記憶にあるチェストを巡って補充する 。

死亡からの自己復旧:

死亡イベントで座標を即保存。リスポーン後、チェストから予備装備を取り出し、死亡地点へ戻ってアイテムを全回収する 。

人間らしさのエミュレーション:

mineflayer-movement による滑らかな歩行 。

待機中に首をランダムに振る、近くのプレイヤーを見つめる、時々独り言をチャットする等の「人間味」を追加 。

ステップ4：運用・更新の完全自動化
GitHubへの自動保存と、プログラムの自動更新を組み込みます。

指示 4: 自動更新とGitHub連携の実装
ボットの稼働を止めず、常に最新の状態に保つスクリプトを scripts/ に作成してください。

自動同期 (deploy.js):

simple-git を使い、ボットが学習した記憶（memory.json）やコードの変更を定期的に自動でGitHubへ push する 。

自動アプデ起動 (run.sh / run.bat):

起動時に git pull を行い、常に最新コードで PM2 を介してボットを立ち上げる 。

PM2 永続化設定:

サーバーやPCの再起動時、OS起動と同時にボットが自動復活する pm2 startup の設定手順を出力してください。

テストコード:

接続、チェスト記憶、移動の3点が正常かを確認する test/full_system.test.js を生成。

仕上げ：エラーチェックと起動ガイド
仕上げの指示
生成したすべてのコードに対して、MacとWindowsの互換性（パス処理等）に誤りがないか最終確認してください。

統合版のItemStackRequest（アイテム移動）の遅延によるパケット拒絶を防ぐための waitForTicks 処理が含まれているか確認してください 。

すべて完了したら、初心者がこのリポジトリを Clone してから、ボットを「一生放置」して動かし続けるための最初のコマンドを日本語で教えてください。

実装に含まれる「便利な追加機能」
資材検索: Web GUIから「ダイヤモンド」と検索すると、ボットが記憶している全チェストの中から、どこに何個あるかを即答します。

自動補充: 建築中にブロックが切れると、ボットは「一番近い資材チェスト」を記憶から呼び出し、自動で取りに行ってから作業を再開します 。

アンチ・キック: サーバー側で長時間放置（AFK）とみなされないよう、微細なランダム移動（Jitter）を常に行います 。

Discord連携（拡張）: 必要であれば、ボットが死亡したりチェストが満杯になった際にDiscordに通知を送る機能もCopilotに頼めばすぐ追加できます 。
